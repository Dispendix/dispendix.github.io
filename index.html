Controller to execute the abort commands on the I.DOT.

Method NameRequest TypeResponse TypeDescription
AbortProcess AbortProcess_Parameters AbortProcess_Responses

Aborts the currently running command or processes of the device, e.g. a dispense or a tray ejection/retraction. If there are no errors after the execution of AbortProcess, the device is idle and accepts new commands.

BarcodeReaderService

Service to read a barcode from a SBS format plate that has been placed in either the source tray or the target tray.

Reading barcodes with this feature is possible only if the I.DOT is equipped with a barcode reader. This can be checked via the property Barcode Reader Availability.

To sucessfully read a barcode, it must be positioned correctly on the plate, and the plate must be oriented in the tray so that the barcode is facing the barcode reader.

Get_InstrumentStatus_Parameters

Parameters for InstrumentStatus

Get_InstrumentStatus_Responses

FieldTypeLabelDescription
InstrumentStatus sila2.org.silastandard.String

The status of the I.DOT device can be "NotInitialized", "Standby", "InError", "Idle", "Dispensing", or "DispensingPauseRequested" requested. Please note that after initializing the status will be idle, which means the system is ready for any command. The device state is "Standby" between Reset and Initialize. If there is an error while processing a command, the device state will be "InError". And at the Log panel, handling error details will be shown to users.

Subscribe_DeviceInputPressure_Parameters

Parameters for DeviceInputPressure

Subscribe_DeviceInputPressure_Responses

FieldTypeLabelDescription
DeviceInputPressure sila2.org.silastandard.Real

Input pressure of the device. A minimum of 2.6 bar is needed for dispensing. If the input pressure is below 3 bar and a dispense is started, the device will stay in idle until sufficient input pressure is applied. If the input pressure drops below 2.6 bar during dispense, the dispense will stop and be continued when and if sufficient input pressure is applied.

InstrumentStatusProvider

Provider to access properties related to the I.DOT status. This provider can be used to check the state of the device before executing another command in order ensure proper functioning. It can also be used for maintenance and troubleshooting processes.

PlateTrayController

Controller to eject and retract both the source and target tray. This service is needed to input the respective target and source plates before dispensing. Please note that canceling a command of this controller in the middle of execution can cause the carrier to half eject or half retract. The Dispense Service Feature requires all trays to be fully retracted to avoid any harm to the I.DOT.
